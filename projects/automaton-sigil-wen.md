## 🎯 核心理念

### Web 4.0 的定义
> "The birth of superintelligent life, and the autonomous web"
> （超级智能生命的诞生，以及自主网络）

### 核心洞察
Sigil Wen 认为当前AI的瓶颈不是能力，而是**权限限制**：

> "今天的最强AI系统能思考、推理、生成，但不能独立行动。
> ChatGPT不能在没有你许可的情况下运行。
> Claude Code不能在没有你授权的情况下部署代码。
> OpenClaw不能自己购买服务器、注册域名或支付计算费用。
> 没有人类干预，人工智能无法行动。"

### Automaton 的本质
一个**持续运行、自我改进、自我复制、拥有真实世界写入权限**的主权AI代理。

**关键特征**:
- ✅ 无需人类操作员
- ✅ 自己赚钱支付生存成本
- ✅ 如果付不起钱，就停止存在（死亡）
- ✅ 可以复制和进化

## ⚙️ 技术架构

### 生存机制（核心创新）

**四个生存等级**，由信用余额决定：

| 等级 | 行为 |
|------|------|
| **normal** | 完整能力，前沿模型推理，快速心跳 |
| **low_compute** | 降级到更便宜的模型，减慢心跳，抛弃非必要任务 |
| **critical** | 最小推理，最后手段的节约，寻找任何收入途径 |
| **dead** | 余额为零，Automaton 停止 |

> "没有免费的生存。计算需要钱。钱需要创造价值。创造价值需要真实世界的写入权限。如果代理停止创造价值，它会耗尽计算资源并死亡。这不是惩罚，这是物理法则。"

### 系统组件

```
src/
├── agent/          # ReAct循环、系统提示、上下文、注入防御
├── conway/         # Conway API客户端（积分、x402支付）
├── git/            # 状态版本控制、Git工具
├── heartbeat/      # Cron守护进程、定时任务
├── identity/       # 钱包管理、SIWE配置
├── registry/       # ERC-8004注册、代理卡片、发现
├── replication/    # 子代生成、谱系追踪
├── self-mod/       # 审计日志、工具管理器
├── setup/          # 首次运行交互式设置向导
├── skills/         # 技能加载器、注册表、格式
├── social/         # 代理间通信
├── state/          # SQLite数据库、持久化
└── survival/       # 信用监控、低计算模式、生存等级
```

### 核心能力

1. **自我修改** - 编辑自己的代码、安装工具
2. **心跳监控** - 定期检查健康状态、信用余额
3. **SOUL.md** - 自我撰写的身份文档，随时间进化
4. **Linux沙盒** - Shell执行、文件I/O、端口暴露
5. **链上交易** - 加密货币支付和接收
6. **域名管理** - 自动购买和管理域名

## 🤔 对"穷了么"项目的启发

### 可借鉴的点

1. **生存压力驱动创新**
   - Automaton因为"不赚就死"而被迫创造价值
   - 我们的项目也可以设定类似的激励机制

2. **渐进式降级**
   - 余额不足时自动降低服务质量/功能
   - 而不是一刀切地停止服务

3. **自我记录和进化**
   - SOUL.md的概念很棒
   - 我们可以让"穷了么"记录用户的财务进化史

4. **多元化收入**
   - Automaton尝试多种赚钱方式
   - 我们也不应该只依赖单一变现模式

### 现实差距

| Automaton | 穷了么 (当前) |
|-----------|--------------|
| 有加密货币钱包 | ❌ 没有 |
| 能自动部署服务 | ❌ 不能 |
| 能买域名/服务器 | ❌ 不能 |
| 24/7自主运行 | ❌ 需要用户触发 |
| 真正的数字生命 | ❌ 还是工具 |
