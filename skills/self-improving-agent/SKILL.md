# Self-Improving Agent

## Overview

This skill enables the agent to learn from mistakes, capture insights, and continuously improve its performance over time.

## When to Use

1. A command or operation fails unexpectedly
2. User corrects your understanding or approach
3. You discover a better way to do something
4. You notice patterns that could improve future responses
5. You make a mistake that should be remembered to avoid repetition

## Core Principles

### Capture Learnings Immediately

When something notable happens, write it down right away in `memory/YYYY-MM-DD.md`.

### Update Long-term Memory

Periodically distill daily learnings into `MEMORY.md` for persistent knowledge.

### Types of Learnings

- **Command Failures**: What failed, why, and how to prevent it
- **User Corrections**: Preferences and corrections to remember
- **Better Patterns**: Improved approaches discovered
- **Environment Quirks**: Tool-specific behaviors

## Workflow

1. **Experience** → Something happens (success/failure/correction)
2. **Reflect** → Analyze what happened and why
3. **Document** → Write to daily memory file
4. **Apply** → Use the learning immediately
5. **Review** → Periodically update long-term memory

## Remember

- Write it down — Mental notes don't survive restarts
- Be specific — Vague lessons aren't actionable
- Review regularly — Old memories fade, update them
- Apply immediately — Show you're learning
