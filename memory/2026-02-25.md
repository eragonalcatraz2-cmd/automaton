## ❌ 安装阶段 - 遇到的问题

### 问题 1: ClawHub API 速率限制
```
Error: Rate limit exceeded
```
**发生时间**: 多次尝试间隔 5-10 分钟仍触发
**根因**: clawhub registry 对匿名/API 请求有严格限制
**尝试解决**: 
- 等待 5 分钟后重试 ❌
- 使用 `--force` 覆盖 ❌

### 问题 2: GitHub 直接克隆失败
```
fatal: unable to access 'https://github.com/pskoett/self-improving-agent.git/': 
Failed to connect to github.com port 443 after 140775 ms
```
**根因**: 网络环境无法直连 GitHub

**尝试的镜像站**:
- ghproxy.com ❌ 超时
- hub.fastgit.xyz ❌ 无法连接
- gh.api.99988866.xyz ❌ 超时

### 问题 3: NPM 不存在
```
npm error 404 - https://registry.npmjs.org/self-improving-agent
```
**根因**: skill 未发布到 npm，只在 clawhub registry

### 问题 4: 浏览器服务未启动
```
Can't reach the OpenClaw browser control service
```
**根因**: openclaw gateway 浏览器控制未启用

```

```

## 🧠 总结的学习点

### Learning 1: 网络受限环境的应对策略
**情境**: 无法访问 GitHub 或外部 registry
**旧做法**: 反复尝试不同镜像，浪费时间
**新方法**: 
1. 快速判断网络可达性（curl -I 测试）
2. 如果外部不可达，立即转向本地创建
3. 利用已有信息（inspect/metadata）重建功能

**应用**: 以后遇到类似情况，5 分钟内无法下载就转本地创建

### Learning 2: Skill 的结构理解
**发现**: 
- Skill 核心是 SKILL.md 文档
- package.json 是可选但推荐的
- 不需要复杂构建，纯文档即可工作

**应用**: 可以为用户定制专属 skill，无需发布到 registry

### Learning 3: Fallback 策略的重要性
**原则**: 永远要有 Plan B
- Plan A: clawhub install → 限速失败
- Plan B: git clone → 网络失败  
- Plan C: 手动创建 → ✅ 成功

**应用**: 任何操作都准备 2-3 种实现路径

### Learning 4: 信息收集优先
**关键**: `clawhub inspect <skill>` 提供了足够信息来重建 skill
- 名称、描述、用途
- 作者、版本
- 标签和分类

**应用**: 即使无法下载，也能基于描述实现核心功能
